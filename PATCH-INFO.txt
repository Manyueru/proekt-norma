Проект «Норма» — функциональный пакет 1.2 (пакет 1.1 после повторного аудита).
Дата подготовки: 2026-07-14.

Основа пакета 1.1 сохранена:
- карта обязательных компетенций;
- тренажёр насмотренности с 6 первыми случаями;
- изучение книг по блокам;
- атлас из 8 методик;
- научная лента PubMed;
- поиск по книгам и методикам;
- удобный выбор тем для дедлайнов;
- резервная копия версии 4.

После повторной сверки с текущим сайтом и кодом исправлено:
- переход из «Карты знаний» теперь открывает все связанные модули, а не ведёт на нефункциональный параметр;
- сохранённый прогресс книги корректно загружается после чтения данных браузера;
- незавершённый каркас книги больше нельзя ошибочно отметить как полностью изученную книгу;
- «Начать заново» в насмотренности действительно удаляет попытку и возвращает карточку в статус новой;
- профессиональный разбор нельзя открыть после одного случайного символа: требуется содержательный минимум;
- дата повторения насмотренности рассчитывается в локальном календаре без сдвига часового пояса;
- счётчик пройденных случаев учитывает только существующие карточки;
- импорт старой резервной копии действительно заменяет данные и очищает разделы, которых в старой версии не было;
- из конспектов убран ещё один список из 251 темы: добавлен поиск и выбор по модулю;
- карточки конспектов доступны с клавиатуры;
- исправлена недопустимая вложенность ссылок и кнопок в основных разделах;
- личный кабинет учитывает насмотренность и изученные книги;
- научная лента получила тайм-аут, корректное кеширование пустого ответа, no-store для ошибок и поддержку переменных NCBI_EUTILS_EMAIL / NCBI_EUTILS_API_KEY;
- добавлено уведомление о правилах NCBI;
- фильтры методик и индикаторы прогресса получили подписи для экранных читалок;
- уточнено название страницы ASHA Communication Milestones;
- год не показывается у постоянно обновляемых ASHA Practice Portal, если официальный год публикации не указан;
- production-команда зафиксирована как `next build --webpack`, то есть совпадает с ранее проверенным режимом сборки.

Важно:
- пакет не отправлялся в Netlify;
- научную ленту всё равно нужно один раз проверить после реального деплоя;
- для корректной идентификации запросов NCBI желательно добавить в Netlify переменную NCBI_EUTILS_EMAIL с рабочим адресом владельца приложения;
- автоматические публикации остаются только библиографической лентой и не меняют учебное содержание без отдельной проверки;
- часть книг и методик остаётся честно помеченной как дополняемый каркас.
