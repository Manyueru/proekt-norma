# Проект «Норма» — версия 2

Персональная образовательная платформа для изучения развития ребёнка, логопедии, нейропсихологии, специальной педагогики и клинической насмотренности.

## Что уже работает

- адаптивный сайт для компьютера и телефона;
- светлая и тёмная темы;
- исправленные страницы учебных тем, наблюдений и клинических задач;
- полный мобильный список разделов;
- главная страница, которая различает нового и активного пользователя;
- статусы тем и подсчёт прогресса;
- личный кабинет;
- регистрация, вход, выход и восстановление пароля через Supabase;
- синхронизация прогресса, конспектов, задач и тестов между устройствами;
- гостевой режим с сохранением данных в браузере;
- быстрые и учебные конспекты;
- просмотр, редактирование, удаление и поиск конспектов;
- сохранение ответов на клинические задачи;
- сохранение результатов мини-тестов;
- поиск по материалам и личным конспектам;
- фильтры модулей, источников и наблюдений.

Учебное наполнение остаётся демонстрационным и должно проходить отдельную научную проверку.

## Что потребуется для личных аккаунтов

Сайт работает без Supabase в гостевом режиме. Для регистрации и синхронизации необходимо один раз создать бесплатный проект Supabase.

### 1. Создать проект Supabase

1. Зарегистрируйтесь в Supabase.
2. Создайте новый проект.
3. Сохраните пароль базы данных в надёжном месте.
4. Дождитесь окончания создания проекта.

### 2. Создать таблицы и правила безопасности

1. Откройте в Supabase раздел **SQL Editor**.
2. Откройте файл `supabase/schema.sql` из этого проекта.
3. Скопируйте его содержимое в SQL Editor.
4. Нажмите **Run**.

Файл создаёт таблицы профилей, прогресса, конспектов, ответов на задачи и результатов тестов. Для всех пользовательских таблиц включён Row Level Security: пользователь видит только собственные записи.

### 3. Взять адрес и ключ проекта

В настройках Supabase найдите раздел API и скопируйте:

- Project URL;
- Publishable key.

Если в старом интерфейсе нет Publishable key, используйте anon key.

### 4. Добавить переменные в Netlify

В Netlify откройте проект и перейдите в настройки переменных окружения. Добавьте:

```text
NEXT_PUBLIC_SUPABASE_URL = адрес проекта Supabase
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY = publishable key
```

Затем запустите **Clear cache and deploy site**.

### 5. Настроить адреса авторизации

В Supabase откройте настройки Authentication → URL Configuration.

Укажите основной адрес Netlify как Site URL, например:

```text
https://название-сайта.netlify.app
```

Добавьте Redirect URLs:

```text
https://название-сайта.netlify.app/**
https://название-сайта.netlify.app/auth/reset-password
```

После этого будут работать подтверждение почты и восстановление пароля.

## Локальный запуск для разработчика

Требуется Node.js 22.

```bash
npm ci
npm run dev
```

Сайт откроется по адресу `http://localhost:3000`.

Проверка типов:

```bash
npm run lint
```

Производственная сборка:

```bash
npm run build
```

## Где хранится контент

Демонстрационные материалы находятся в папке `data`:

- `topics.json` — учебные темы;
- `sources.json` — источники;
- `observations.json` — карточки наблюдений;
- `cases.json` — клинические задачи;
- `glossary.json` — словарь;
- `videos.json` — видео.

Контент отделён от компонентов сайта. Новую тему можно добавить в `data/topics.json`, используя структуру существующих записей.

---

## Где хранятся пользовательские данные

### Без входа

В `localStorage` текущего браузера. Они не появятся на другом устройстве и могут исчезнуть после очистки браузера. В гостевом режиме доступен экспорт резервной копии.

### После входа

В таблицах Supabase. Доступ ограничен политиками Row Level Security. Секретный service role key в клиентском приложении не используется и не должен добавляться в репозиторий.

## Публикация через Netlify

1. Загрузите содержимое проекта в GitHub.
2. Подключите репозиторий к Netlify.
3. Netlify определит Next.js автоматически.
4. Build command: `npm run build`.
5. После подключения Supabase добавьте переменные окружения.
6. Запустите новую публикацию с очисткой кэша.

Файл `netlify.toml` уже фиксирует совместимую версию Node и npm.

## Ограничения текущей версии

- учебная информация пока демонстрационная;
- нет административной панели для добавления материалов через браузер;
- нет платных функций, комментариев и социальных возможностей;
- гостевые данные не переносятся в аккаунт автоматически;
- две умеренные записи `npm audit` связаны с PostCSS в цепочке Next.js; высоких и критических уязвимостей нет. Принудительный откат зависимостей не применяется.
