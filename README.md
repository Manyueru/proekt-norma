# Проект «Норма» — предпросмотр полного учебного каркаса

Персональная образовательная платформа для изучения развития ребёнка, логопедии, нейропсихологии, специальной педагогики и клинической насмотренности.

## Что работает

- адаптивный сайт для компьютера и телефона;
- светлая и тёмная темы;
- 22 учебных модуля и 251 тема с логопедией как основной траекторией;
- фильтры по направлению, готовности содержания и учебному статусу;
- отдельный безопасный режим для тем-каркасов без неподтверждённого текста;
- источники, видеотека, насмотренность, клинические задачи и словарь;
- локальный личный кабинет со статистикой;
- статусы тем и подсчёт прогресса;
- быстрые и учебные конспекты с редактированием;
- сохранение ответов на клинические задачи и результатов мини-тестов;
- общий поиск;
- раздел «Подготовка к экзаменам»;
- добавление экзаменов и экзаменационных вопросов;
- статусы готовности вопросов;
- режим случайного вопроса и пробного билета;
- примерный темп подготовки к экзамену;
- раздел «Учебные задачи» с дедлайнами;
- автоматическое предупреждение о приближении срока;
- подзадачи, приоритеты и связь задачи с экзаменом;
- блок ближайших дедлайнов и ближайшего экзамена на главной странице;
- экспорт и импорт локальной резервной копии.

В этой версии опубликован полный каркас из 22 модулей и 251 темы. Четыре темы заполнены и проверены для предпросмотра; остальные отображают только будущую структуру без неподтверждённого учебного текста.

## Важное ограничение текущей версии

Экзамены, вопросы, дедлайны и другие персональные данные сейчас сохраняются в `localStorage` текущего браузера.

Это означает:

- данные на компьютере и телефоне пока не синхронизируются;
- после очистки данных браузера записи могут исчезнуть;
- необходимо периодически сохранять резервную копию;
- синхронизация будет подключена позднее, после выбора сервиса, стабильно доступного пользователям из России.

Код экспериментальной интеграции Supabase сохранён в проекте, но сервис не требуется для работы локального кабинета и сейчас не рекомендуется как обязательная основа платформы.

## Локальный запуск для разработчика

Требуется Node.js 22.

```bash
npm ci
npm run dev
```

Сайт откроется по адресу `http://localhost:3000`.

Проверка типов:

```bash
npm run lint
```

Производственная сборка:

```bash
npm run build
```

## Где хранится учебный контент

Демонстрационные материалы находятся в папке `data`:

- `modules.json` — описание 22 учебных модулей;
- `topics.json` — 251 учебная тема и статус готовности содержания;
- `sources.json` — источники;
- `observations.json` — карточки наблюдений;
- `cases.json` — клинические задачи;
- `glossary.json` — словарь;
- `videos.json` — видео.

## Где хранятся пользовательские данные

Локальные данные хранятся в браузере под ключами `norma:v2:*` и `norma:v3:*`.

Резервная копия версии 3 включает:

- прогресс тем;
- конспекты;
- ответы на клинические задачи;
- результаты тестов;
- учебные задачи и дедлайны;
- экзамены и экзаменационные вопросы.

## Публикация через Netlify

1. Загрузите изменённые файлы в существующий репозиторий GitHub.
2. Netlify автоматически запустит новую сборку.
3. Если сборка не началась, выберите **Trigger deploy → Clear cache and deploy site**.
4. Build command: `npm run build`.

Файл `netlify.toml` уже содержит настройки сборки.

## Ограничения предпросмотра

- нет синхронизации между устройствами;
- нет календарного вида дедлайнов;
- нет голосовой записи ответов;
- нет автоматической проверки устного ответа;
- нет импорта экзаменационных вопросов из Word или PDF;
- нет административной панели для учебного контента;
- две умеренные записи `npm audit` относятся к цепочке PostCSS; высоких и критических уязвимостей нет.
